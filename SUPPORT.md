## Getting Help

### For Bugs and Issues

If you've found a bug or have a technical issue:

1. Search [existing issues](https://github.com/your-username/state-ski-resort-directory/issues) to see if it's already reported
2. If not found, [create a new issue](https://github.com/your-username/state-ski-resort-directory/issues/new/choose) using our bug report template
3. Include as much detail as possible:
   - Steps to reproduce
   - Expected vs actual behavior
   - Browser and OS information
   - Screenshots if applicable

### For Feature Requests

Have an idea for a new feature?

1. Check [existing feature requests](https://github.com/your-username/state-ski-resort-directory/issues?q=label%3Aenhancement)
2. [Submit a feature request](https://github.com/your-username/state-ski-resort-directory/issues/new/choose) using our template
3. Describe the problem you're solving and your proposed solution

### For Questions and Discussions

For general questions, ideas, or community discussions:

- Check the documentation first
- Search existing issues for similar questions
- Open a new issue with the "question" label if needed

## Security Issues

**Do not report security vulnerabilities through public issues.**

Please see our [Security Policy](SECURITY.md) for responsible disclosure procedures.

## Response Times

We aim to respond to:

| Type | Target Response |
|------|-----------------|
| Security issues | Within 48 hours |
| Bug reports | Within 5 business days |
| Feature requests | Within 10 business days |
| General questions | Best effort |

Please be patient - this is a community-maintained project.

## What NOT to Ask Here

To keep support channels focused, please avoid:

- Questions already answered in documentation
- Requests for general programming help unrelated to this project
- Issues with third-party services (Supabase, Netlify, etc.) - contact their support
