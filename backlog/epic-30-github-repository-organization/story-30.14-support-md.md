# Story 30.14: Add SUPPORT.md with Help Channels

## Description

Create SUPPORT.md to direct users to appropriate help channels, distinguishing between documentation, community support, and issue reporting.

## Acceptance Criteria

- [ ] `SUPPORT.md` file exists at repository root
- [ ] Documentation links included
- [ ] Issue reporting guidance provided
- [ ] Discussion channels listed
- [ ] Clear distinction between support types

## Technical Details

### SUPPORT.md Template

```markdown
# Support

Thank you for using Ski Resort Directory! This document provides guidance on how to get help.

## Documentation

Before seeking support, please check our documentation:

- [README.md](README.md) - Project overview and quick start
- [CLAUDE.md](CLAUDE.md) - Development commands and architecture
- [docs/](docs/) - Detailed documentation
  - [Architecture](docs/architecture.md)
  - [Data Model](docs/data-model.md)
  - [API Reference](docs/api-reference.md)
  - [Development Guide](docs/development.md)

## Getting Help

### For Bugs and Issues

If you've found a bug or have a technical issue:

1. Search [existing issues](https://github.com/YOUR_USERNAME/state-ski-resort-directory/issues) to see if it's already reported
2. If not found, [create a new issue](https://github.com/YOUR_USERNAME/state-ski-resort-directory/issues/new/choose) using our bug report template
3. Include as much detail as possible (steps to reproduce, expected vs actual behavior, screenshots)

### For Feature Requests

Have an idea for a new feature?

1. Check [existing feature requests](https://github.com/YOUR_USERNAME/state-ski-resort-directory/issues?q=is%3Aissue+label%3Aenhancement)
2. [Submit a feature request](https://github.com/YOUR_USERNAME/state-ski-resort-directory/issues/new/choose) using our template
3. Describe the problem you're solving and your proposed solution

### For Questions and Discussions

For general questions, ideas, or community discussions:

1. Use [GitHub Discussions](https://github.com/YOUR_USERNAME/state-ski-resort-directory/discussions) (if enabled)
2. Check the [Q&A category](https://github.com/YOUR_USERNAME/state-ski-resort-directory/discussions/categories/q-a) for existing answers

## Contributing

Interested in contributing? See our [Contributing Guide](CONTRIBUTING.md).

## Security Issues

**Do not report security vulnerabilities through public issues.**

Please see our [Security Policy](SECURITY.md) for responsible disclosure procedures.

## Response Times

We aim to respond to:

| Type | Target Response |
|------|-----------------|
| Security issues | 48 hours |
| Bug reports | 5 business days |
| Feature requests | 10 business days |
| General questions | Best effort |

Please be patient - this is a community project maintained by volunteers.

## What NOT to Ask Here

To keep support channels focused, please avoid:

- Questions answered in documentation
- Requests for general programming help
- Issues unrelated to this project

Thank you for your understanding and for being part of our community!
```
